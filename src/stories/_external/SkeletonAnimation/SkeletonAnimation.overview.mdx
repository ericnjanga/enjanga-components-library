  <h2 id="purpose-what-is-it" style={{ position: 'relative' }}>üìå Purpose / What it is?</h2>  
  <p>The <code>SkeletonAnimation</code> component provides a visual placeholder while content is loading. It mimics the shape and structure of text or list elements using animated skeleton blocks, giving users a sense of layout before real data is available.</p>

  <Primary />

<div style={{ position: 'relative', paddingTop: '50px', marginTop: '-50px', zIndex: '1', marginBottom: '60px' }}>
  <h2 id="when-to-use" style={{ position: 'relative' }}>‚úÖ When to Use</h2> 
  <ul data-start="451" data-end="757">
  <li data-start="451" data-end="567">
  <p data-start="453" data-end="567">When content is being fetched from an API (e.g., Contentful, REST, GraphQL) and you want to avoid layout shifts.</p>
  </li>
  <li data-start="568" data-end="650">
  <p data-start="570" data-end="650">To indicate loading states for <strong data-start="601" data-end="613">headings</strong>, <strong data-start="615" data-end="628">body text</strong>, or <strong data-start="633" data-end="647">list items</strong>.</p>
  </li>
  <li data-start="651" data-end="757">
  <p data-start="653" data-end="757">When you want to provide a smoother perceived performance by showing placeholders instead of spinners.</p>
  </li>
  </ul>
</div>



<div style={{ position: 'relative', paddingTop: '50px', marginTop: '-50px', zIndex: '1', marginBottom: '60px' }}>
  <h2 id="when-not-to-use" style={{ position: 'relative' }}>‚ùå When not to use</h2>
  <ul data-start="788" data-end="1083">
  <li data-start="788" data-end="887">
  <p data-start="790" data-end="887">If loading times are very short (milliseconds) ‚Äî in such cases a skeleton may feel unnecessary.</p>
  </li>
  <li data-start="888" data-end="976">
  <p data-start="890" data-end="976">For non-visual content (like background tasks) where the user doesn‚Äôt need feedback.</p>
  </li>
  <li data-start="977" data-end="1083">
  <p data-start="979" data-end="1083">When exact element dimensions are unknown (as skeletons are designed to mimic expected content sizes).</p>
  </li>
  </ul>
</div>
