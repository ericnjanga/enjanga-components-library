  <h2 id="purpose-what-is-it" style={{ position: 'relative' }}>üìå Purpose / What it is?</h2>  
  <div>The <code>SmartText</code> component is a lightweight wrapper for rendering text content. It intelligently decides whether to display plain text (<code>&lt;p&gt;</code>) or render structured Contentful rich text via the <code>CMSRichText</code> component.</div>
  <div>It ensures consistent text rendering across the design system while supporting both simple and complex text inputs.</div>

<div style={{ position: 'relative', paddingTop: '50px', marginTop: '-50px', zIndex: '1', marginBottom: '60px' }}>
  <h2 id="property-validation" style={{ position: 'relative' }}>Property validation</h2> 
  <div>The <code>SmartText</code> component accepts <strong>either</strong> a <code>plainText</code> prop or a <code>richText</code> prop, but <strong>never both at the same time</strong>.</div>

  <ul data-start="440" data-end="678">
  <li data-start="440" data-end="571">
  <p data-start="442" data-end="462">‚úÖ <strong data-start="444" data-end="459">Valid cases</strong>:</p>
  <ul data-start="465" data-end="571">
  <li data-start="465" data-end="515">
  <p data-start="467" data-end="515">Pass only <code data-start="477" data-end="488">plainText</code> (simple string of text).</p>
  </li>
  <li data-start="518" data-end="571">
  <p data-start="520" data-end="571">Pass only <code data-start="530" data-end="540">richText</code> (Contentful Rich Text JSON).</p>
  </li>
  </ul>
  </li>
  <li data-start="573" data-end="678">
  <p data-start="575" data-end="596">‚ùå <strong data-start="577" data-end="593">Invalid case</strong>:</p>
  <ul data-start="599" data-end="678">
  <li data-start="599" data-end="678">
  <p data-start="601" data-end="678">Passing both <code data-start="614" data-end="625">plainText</code> and <code data-start="630" data-end="640">richText</code> at once will throw a runtime error.</p>
  </li>
  </ul>
  </li>
  </ul>

  <div>This validation is enforced by the helper function <code>smartTextPropsValidation()</code>, which checks the props on render and throws a descriptive error if the rule is violated.</div>

  <div>If neither prop is provided, the component gracefully falls back to displaying a <strong>skeleton animation placeholder</strong>, ensuring the layout remains consistent.</div>

</div>
